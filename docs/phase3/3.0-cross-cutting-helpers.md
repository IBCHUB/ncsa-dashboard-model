## Fixes applied (commit `a578e50`)

### Fix 1 ‚Äî Phantom date fields removed (CRITICAL)

```diff
 WAREHOUSE_TIME_FIELDS: dict[str, list[str]] = {
     "observed": ["event_time", "first_seen", "last_seen"],
     "processed": ["processed_at", "created_at", "collect_time"],
     "published": ["published_at"],
-    "changed": ["revoked_at", "last_shared_at", "updated_at"],
+    "changed": ["last_shared_at", "action_updated_at"],
 }

 PYTHON_FILTER_FIELDS: dict[str, list[str]] = {
     "observed": ["event_time", "first_seen", "last_seen", "observation_date"],
     "processed": ["processed_at", "created_at", "collect_time"],
     "published": ["published_at"],
-    "changed": ["revoked_at", "last_shared_at", "updated_at"],
+    "changed": ["last_shared_at", "action_updated_at"],
 }
```

**Evidence:** `es_mapping_warehouse.json` ‚Äî `revoked_at` and `updated_at` keys absent.

**Impact:** previously any caller passing `time_mode="changed"` got a date range over three OR'd fields, two of which never existed in ES. ES would silently treat them as missing, so the range only applied via `last_shared_at`. Now explicit.

### Fix 5 ‚Äî `warehouse_eligible_only` parameter (HIGH)

```diff
 def _warehouse_summary_stats(
     start_date: Optional[str],
     end_date: Optional[str],
     time_mode: str = TIME_MODE_PROCESSED,
     *,
     sources: Optional[List[str]] = None,
     threat_types: Optional[List[str]] = None,
     severities: Optional[List[str]] = None,
     ioc_types: Optional[List[str]] = None,
+    warehouse_eligible_only: bool = True,
 ) -> Dict[str, Any]:
```

**Why:** the previous implementation hard-coded `warehouse_eligible_only=True` (via the default of `_warehouse_search_filters`), silently AND-filtering on `warehouse_eligible=true` and dropping the 372,389 rejected docs even when callers wanted total counts.

**Default unchanged** ‚Äî preserves existing behavior; only opens the door for explicit opt-out.

### Fix 7 ‚Äî Dead Thailand filter clauses (MED)

```diff
 "thailand_threat": {
-    "filter": {
-        "bool": {
-            "should": [
-                {"term": {"geo_country": "Thailand"}},
-                {"term": {"geo_country": "TH"}},
-                {"term": {"geo_country": "thailand"}},
-            ],
-            "minimum_should_match": 1,
-        }
-    },
+    "filter": {"term": {"geo_country": "TH"}},
 },
```

**Evidence:** `warehouse_field_distribution.json` ‚Äî `geo_country` contains only ISO-2 codes (`IN, US, BR, CN, ‚Ä¶`) plus the literal string `None` and MISSING. No `Thailand` value ever appears.
