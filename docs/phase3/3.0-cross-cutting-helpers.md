## ข้อสังเกต / Data quality

| # | ประเด็น | ผลต่อหน้าเว็บ |
|---|--------|---------------|
| 1 | `severity` และ `ai_severity` ในข้อมูลปัจจุบัน: distribution เหมือนกัน 99.99% (low 11M / medium 40K / critical 8.6K / high 1 doc) เพราะ 99.99% เป็น single-source Cyberint | การเปลี่ยนเป็น `ai_severity` จะไม่เปลี่ยนตัวเลขที่เห็นตอนนี้ แต่ถูกต้อง semantic และรองรับ multi-source ในอนาคต |
| 2 | `tlp = amber` 100% ของข้อมูล | TLP gating (Phase 2) ทำงานถูก แต่ยังไม่มีข้อมูล non-amber ให้กรองจริง |
| 3 | `geo_country` 99.4% เป็น "None"/MISSING — ค่าประเทศจริงแค่ 0.4% | "Top Attack Origins" บน Executive/Operations แสดงจาก data ส่วนน้อยมาก — ไม่ใช่ bug แต่เป็น data coverage |
| 4 | `target_sector` 97.9% MISSING — ค่าจริงแค่ 2% | "Target Sectors" chart มีข้อมูลแค่ 2% — coverage ต่ำ |
| 5 | `source_name` 99.99% เป็น cyberint_iocs | "Top Intelligence Sources" = Cyberint แทบจะอย่างเดียว — chart ไม่ informative |
| 6 | `ai_threat_types` ปนกัน Title Case (Malware, Phishing) กับ snake_case (malware_payload, phishing_website) | "Threat Types" chart อาจแสดง 2 แท่งสำหรับความหมายเดียวกัน (e.g., Phishing 894 + phishing_website 364K) |
| 7 | `action_status` มีแค่ open กับ MISSING (ไม่มี closed/in_progress) | Action Center workflow ยังไม่ได้ใช้งานจริง |
| 8 | `reviewed_at` field มีอยู่ใน mapping แต่ไม่มี doc ไหนใส่ค่าเลย | ฟีเจอร์ review workflow ยังไม่ทำงาน |
| 9 | `victim_country` / `target_country` ไม่มีใน ES warehouse mapping — Cyberint raw feed มีแค่ 9 fields ไม่มี geo target | "Thailand Threat" badge ใช้ best-effort text match แทน field โดยตรง — แก้ได้จริงต้องรอ Phase 4+ (ingestion pipeline เพิ่ม target_country) |

## สถาปัตยกรรม severity (หลัง BUG-3.0-8)

```
scorer.py
  └── calculate_risk_score()
        ├── อ่าน severity จาก 6 factors
        └── เขียน doc.ai_severity = "critical/high/medium/low/clean"

pipeline_documents.py
  ├── เขียน doc.severity     = raw Cyberint score (preserved)
  └── เขียน doc.ai_severity = AI-computed label  ← dashboard ใช้อันนี้

dashboard_router.py  (หลังแก้ไข)
  ├── ES warehouse queries  → field "ai_severity"  ✅
  ├── _severity_filters_config() default → "ai_severity"  ✅
  ├── _ai_severity(doc)  → doc.get("ai_severity") or doc.get("severity")  ✅
  └── ES datalake queries   → field "severity" (datalake ไม่มี ai_severity)  ✅
```
