## รายละเอียดการแก้ (commit `a578e50`)

### Fix 1 — ลบ phantom date fields (CRITICAL)

```diff
 WAREHOUSE_TIME_FIELDS: dict[str, list[str]] = {
     "observed": ["event_time", "first_seen", "last_seen"],
     "processed": ["processed_at", "created_at", "collect_time"],
     "published": ["published_at"],
-    "changed": ["revoked_at", "last_shared_at", "updated_at"],
+    "changed": ["last_shared_at", "action_updated_at"],
 }

 PYTHON_FILTER_FIELDS: dict[str, list[str]] = {
     "observed": ["event_time", "first_seen", "last_seen", "observation_date"],
     "processed": ["processed_at", "created_at", "collect_time"],
     "published": ["published_at"],
-    "changed": ["revoked_at", "last_shared_at", "updated_at"],
+    "changed": ["last_shared_at", "action_updated_at"],
 }
```

**หลักฐาน:** `es_mapping_warehouse.json` — search หา `revoked_at` และ `updated_at` ไม่เจอ (key ไม่อยู่)

**ผลกระทบเดิม:** caller ใดที่ส่ง `time_mode="changed"` จะได้ date range OR ของ 3 field ซึ่ง 2 ใน 3 ไม่เคยมีจริง ES treat เป็น missing เงียบๆ → filter ทำงานจริงผ่าน `last_shared_at` ตัวเดียว ตอนนี้ explicit แล้ว

### Fix 5 — เพิ่ม `warehouse_eligible_only` parameter (HIGH)

```diff
 def _warehouse_summary_stats(
     start_date: Optional[str],
     end_date: Optional[str],
     time_mode: str = TIME_MODE_PROCESSED,
     *,
     sources: Optional[List[str]] = None,
     threat_types: Optional[List[str]] = None,
     severities: Optional[List[str]] = None,
     ioc_types: Optional[List[str]] = None,
+    warehouse_eligible_only: bool = True,
 ) -> Dict[str, Any]:
```

**ทำไม:** การ implement เดิม hard-code `warehouse_eligible_only=True` (ผ่าน default ของ `_warehouse_search_filters`) → AND filter `warehouse_eligible=true` เงียบๆ และทิ้ง 372,389 rejected docs ทั้งๆ ที่ caller บางตัวอยากนับ total รวม

**Default เดิม** — preserve behavior เดิม เปิดประตูให้ opt-out ได้เท่านั้น

### Fix 7 — ลบ Thailand clauses ตายแล้ว (MED)

```diff
 "thailand_threat": {

-                {"term": {"geo_country": "Thailand"}},
```
