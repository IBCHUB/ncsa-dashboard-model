## ข้อสังเกต / Data quality

| # | ประเด็น | ผลต่อหน้าเว็บ |
|---|--------|---------------|
| 1 | **"Attacker" column hardcoded "Anonymous"** ใน frontend (line 748) | ทุก row บน IOC table แสดง "Anonymous" — ไม่มีข้อมูลจริง (backend ไม่ส่ง field นี้, `ai_threat_actors` data quality ต่ำ) — เป็น UI design decision ห้ามแก้ frontend |
| 2 | "Threat Types" column ปน Title Case + snake_case | UI badge อาจขึ้น 2 badges สำหรับความหมายเดียวกัน (e.g., "Phishing" + "phishing_website") — Phase 1 ingestion territory |
| 3 | IOC Data Analytics tab = duplicate ของ TI IOC Summary (3.3) | maintenance ง่ายขึ้น แต่ user ที่ navigate 2 หน้าจะเห็นข้อมูลเดียวกัน |
| 4 | Threat Hunting graph nodes "Actor" แทบไม่มีข้อมูล | data quality ของ `ai_threat_actors` ต่ำ — graph จะแสดง threat type / infrastructure / malware เป็นหลัก |
| 5 | **Threat Hunting + IOC Detail: `_fetch_datalake_by_indicators` ไม่มี cap** (line 7450, 7508) | popular IOC ที่มี 100K+ docs ใน datalake → slow page load + memory pressure — **Phase 2 performance follow-up** (defer) |
