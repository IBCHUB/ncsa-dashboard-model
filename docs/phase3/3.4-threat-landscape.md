### ก่อนแก้ vs หลังแก้

**ก่อนแก้** — user เห็น "Intelligence Sources" list:
```
1. cyberint_iocs-2025.09.03      1,049,096
2. cyberint_iocs-2025.09.02        889,575
3. cyberint_iocs-2025.09.09        744,026
4. cyberint_iocs-2025.09.04        738,605
5. cyberint_iocs-2025.09.10        720,495
6. cyberint_iocs-2025.09.05        711,260
... (10 entries ทั้งหมดเป็น cyberint daily indices)
```

**หลังแก้** — user เห็น:
```
1. cyberint_iocs       48,397,382  (รวมทุก daily index)
2. (warehouse sources อื่นๆ ถ้ามี)
```

| # | Severity | Label ที่กระทบ | สถานะ |
|---|----------|---------------|------|
| BUG-3.0-9 | MED | Intelligence Sources | ✅ ตอนนี้แก้แล้วผ่าน BUG-3.4-1 (เพิ่ม normalization) — แต่ root cause (datalake aggregate `_index` ไม่ใช่ `source_name`) ยังคงอยู่ |

## ข้อสังเกต / Data quality

| # | ประเด็น | ผลต่อหน้าเว็บ |
|---|--------|---------------|
| 1 | datalake `source_name` field **100% MISSING** (48.4M docs ทุก doc) | backend จำเป็นต้อง agg `_index` แทน — ไม่มีทางอื่น |
| 2 | datalake ใช้ daily-rotated indices ชื่อ `cyberint_iocs-YYYY.MM.DD` (47+ indices ในช่วง 1 เดือนล่าสุด) | ก่อนแก้: leak ชื่อรายวันขึ้น UI; หลังแก้: consolidate ถูกต้อง |
| 3 | "Attack Origins" filter ชื่อประเทศ "thailand"/"th"/"unknown" ออก (line 5935) | semantic: หน้านี้แสดง origin **ที่ไม่ใช่ Thailand** (เพราะหน้านี้ดูภัยที่เข้าไทย) — ตั้งใจเช่นนี้ |
| 4 | Threat Landscape ใช้ `time_mode=observed` (event_time) — ต่างจาก Executive/Operations ที่ใช้ processed | totals 7d window จะต่างกัน — ผู้ใช้ที่เปรียบเทียบหน้าต่อหน้าอาจสับสน |
| 5 | data ปัจจุบัน Threat Actors มักว่าง (`ai_threat_actors` array แทบไม่ถูก populate) | "Threat Actors" ranking อาจว่างหรือมีน้อย — data quality |
| 6 | warehouse `source_name` มี 99.99% เป็น `cyberint_iocs` | warehouse contribution ใน "Intelligence Sources" จะมีแค่ 1 entry |
