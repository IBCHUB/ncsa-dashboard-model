## ข้อสังเกต / Data quality

| # | ประเด็น | ผลต่อหน้าเว็บ |
|---|--------|---------------|
| 1 | **"Worldwide Threat" ≠ "ทั่วโลก" จริงๆ** — รวม doc ที่ไม่รู้ประเทศด้วย (99.4% ของ data) | label สื่อความหมาย "worldwide" แต่ความจริงคือ "all threats" (Thailand subset + 99% docs ไม่รู้ประเทศ) — ไม่ใช่ bug แต่ user ที่อ่าน label อาจเข้าใจผิด |
| 2 | severity = **high มีแค่ 1 doc ใน 11M docs** | "High Active" KPI card จะแสดงตัวเลขใกล้ 0 เกือบตลอด — ไม่ใช่ bug แต่ data limitation |
| 3 | `geo_country` 99.4% เป็น `"None"`/MISSING — ค่าประเทศจริงแค่ 0.4% (40K docs จาก 11M) | Threat Map + "Top Attack Origins" + Thailand Threat — ทั้งหมดอ้างอิงข้อมูลส่วนน้อยมาก (coverage ต่ำ) |
| 4 | `target_sector_name` 97.9% MISSING — ค่า sector จริงแค่ 2% | "Most Target Sector" + Sector treemap — coverage ต่ำ |
| 5 | `ai_threat_types` ปนกัน Title Case (Malware, Phishing) กับ snake_case (malware_payload, phishing_website) | "Threat Volume & Severity Analysis" treemap แสดง 2 แท่งสำหรับความหมายเดียวกัน — Phase 1 ingestion territory |
| 6 | `source_name` 99.99% เป็น cyberint_iocs | "High Confidence Sources" จะนับได้ ≤ 1 (Cyberint) เสมอ |
| 7 | "IOC Active" ใช้ cardinality ที่ `precision_threshold=40000` | ตัวเลขที่แสดงมีความคลาดเคลื่อนเล็กน้อย (ปกติ ES cardinality error rate < 1% สำหรับ threshold 40000) |
