## 📐 Thailand Threat Level formula

> ระดับเตือนภัย **คนละสเกลกับ IOC severity** — อ่านตรงนี้ก่อนแก้ไข

| สิ่งที่วัด | IOC Severity | Thailand Threat Level |
|----------|-------------|----------------------|
| คืออะไร | ระดับอันตรายของ IOC แต่ละตัว | ระดับภาพรวมภัยคุกคามทั้งหมดในช่วงเวลา |
| ค่าที่มี | critical / high / medium / low / clean | วิกฤต / ยกระดับ / เฝ้าระวัง / ต่ำ |
| ภาษาอังกฤษ | critical / high / medium / low / clean | critical / elevated / guarded / low |
| อยู่ใน field | `ai_severity` (ใน ES doc) | `threat_level.level` (ใน response) |
| คำนวณอย่างไร | AI scorer.py (6 factors per IOC) | `_build_threat_level_from_aggregations()` (composite) |

### สูตร composite (ใน `_build_threat_level_from_aggregations`)

```
score (0–100) =
  (Volume spike)  × 30%   ← latest bucket / baseline average ratio
  (Severity ratio) × 25%  ← (critical + high docs) / total docs
  (Sector impact)  × 25%  ← จำนวน sector ที่มี high/critical IOC (×1.5 ถ้ามี CII sector)
  (Actor activity) × 20%  ← จำนวน named threat actor ที่ตรวจพบ

score ≥ 76 → "วิกฤต" (critical)
score ≥ 51 → "ยกระดับ" (elevated)
score ≥ 26 → "เฝ้าระวัง" (guarded)
score < 26 → "ต่ำ" (low)
```

ตัวอย่าง: ถ้าวันนี้มี IOC เข้ามา 3× ค่าปกติ, สัดส่วน critical+high 30%, กระทบ 2 sector, ไม่พบ named actor
→ score ≈ 100×0.30 + 80×0.25 + 60×0.25 + 10×0.20 = 30+20+15+2 = **67** → **ยกระดับ**

**ข้อสังเกตจาก spot-check:**
1. ทุก KPI คำนวณถูกต้องตาม formula
