# TCTI — ภาพรวมระบบ Cyber Threat Intelligence

Last updated: 2026-05-15

---

## ระบบนี้ทำอะไร?

**TCTI (Thailand Cyber Threat Intelligence)** เป็นแพลตฟอร์ม AI ที่รวบรวมข้อมูลภัยคุกคามทางไซเบอร์จากหลายแหล่ง นำมาวิเคราะห์อัตโนมัติ แล้วแสดงผลเป็น Dashboard ให้ผู้บริหารและเจ้าหน้าที่ SOC ใช้ตัดสินใจได้ทันท่วงที

## Pipeline ทำงานอย่างไร?

Pipeline คือกระบวนการอัตโนมัติที่นำข้อมูลดิบมาแปลงเป็นข้อมูลข่าวกรองที่พร้อมใช้งาน แบ่งเป็น 4 ขั้นตอน:

### ขั้นตอนที่ 1 — Normalize (ทำข้อมูลให้เป็นรูปแบบเดียวกัน)

ข้อมูลจากแต่ละแหล่งมีรูปแบบต่างกัน เช่น MISP ส่งมาแบบหนึ่ง Cyberint ส่งมาอีกแบบ ระบบจะแปลงทุกรูปแบบให้เป็นโครงสร้างมาตรฐานเดียวกัน (Canonical IOC) ข้อมูลที่ผิดรูปแบบจะถูกกักเก็บ (quarantine) ไม่ให้เข้าสู่ระบบ

### ขั้นตอนที่ 2 — Classify (จำแนกประเภทภัยคุกคาม)

ระบบจำแนกว่าข้อมูลแต่ละชิ้นเป็นภัยคุกคามประเภทใด โดยใช้ 2 วิธี:

| วิธี | ใช้เมื่อ | ตัวอย่าง |
|------|---------|---------|
| **Rule-Based** (กฎจากแหล่งข้อมูล) | ข้อมูลจาก IOC Feed, MISP ที่มี metadata ชัดเจน | tag = "phishing_website" → Phishing |
| **AI/ML** (โมเดล DeBERTa) | ข้อความข่าว, รายงาน, advisory ที่ต้องตีความ | "attackers deployed ransomware..." → Ransomware |

นอกจากประเภทภัย ระบบยังระบุ:
- **Threat Actor** — กลุ่มผู้โจมตี (เช่น APT28, Lazarus Group)
- **MITRE ATT&CK** — เทคนิคการโจมตีตามมาตรฐานสากล

### ขั้นตอนที่ 3 — Score (ให้คะแนนความเสี่ยง)

ระบบคำนวณคะแนนความเสี่ยง 0-100 จากหลายปัจจัย:

| ปัจจัย | หลักการ |
|--------|---------|
| จำนวนแหล่งยืนยัน | IOC ที่หลายแหล่งรายงานตรงกัน = น่าเชื่อถือกว่า |
| คุณภาพแหล่งข้อมูล | VirusTotal, MISP ได้คะแนนสูงกว่าข่าวทั่วไป |
| คำสำคัญเชิงภัย | มีคำว่า "ransomware", "zero-day" = เพิ่มคะแนน |
| ความผิดปกติของโดเมน | ชื่อโดเมนสุ่ม (DGA) หรือจดทะเบียนใหม่ = น่าสงสัย |
| ประเภทภัยจาก AI | Ransomware ร้ายแรงกว่า Spam |
| กลุ่มผู้โจมตี | APT ที่เป็นที่รู้จักได้คะแนนสูง |
| กลุ่มเป้าหมาย | โจมตีโครงสร้างพื้นฐานสำคัญ (CII) = คูณคะแนนเพิ่ม |

คะแนนแปลงเป็นระดับความรุนแรง:

| คะแนน | ระดับ |
|--------|-------|
| 75-100 | **Critical** (วิกฤต) |
| 50-74 | **High** (สูง) |
| 25-49 | **Medium** (ปานกลาง) |
| 1-24 | **Low** (ต่ำ) |
| 0 | **Clean** (ปลอดภัย) |

### ขั้นตอนที่ 4 — Validate (ตรวจสอบคุณภาพ)

ระบบตรวจสอบว่าข้อมูลมีคุณภาพเพียงพอก่อนเข้า Warehouse:
- IOC ระดับ Critical ต้องมีแหล่งยืนยันอย่างน้อย 2 แหล่ง
- ข้อมูลจากข่าวอย่างเดียว (ไม่มีแหล่งอื่นยืนยัน) จะจำกัดไว้สูงสุดแค่ High
- ข้อมูลคุณภาพต่ำจะถูก reject ไม่แสดงใน Dashboard

## Dashboard แสดงอะไรบ้าง?

### Executive Dashboard — สำหรับผู้บริหาร

| ตัวเลข | ความหมาย |
|--------|---------|
| Thailand Threat Level | ระดับภัยคุกคามรวมของประเทศ (LOW / GUARDED / ELEVATED / CRITICAL) |
| Total Threats | จำนวนภัยคุกคามทั้งหมดในช่วงเวลาที่เลือก |
| Critical Threats | จำนวนภัยระดับวิกฤตที่ต้องดำเนินการเร่งด่วน |
| Worldwide Threats | จำนวนภัยคุกคามทั่วโลก |
| Predicted Next 24h | การพยากรณ์ปริมาณภัยใน 24 ชั่วโมงข้างหน้า (AI Forecast) |
| Attack Volume Trend | แนวโน้มปริมาณการโจมตีรายวัน |
| Attack Origins Map | แผนที่แสดงต้นทางการโจมตี |
| Threat Volume Treemap | สัดส่วนประเภทภัยคุกคาม |
| Target Sector Analysis | กลุ่มเป้าหมายที่ถูกโจมตีมากที่สุด |

### Threat Intelligence — วิเคราะห์ภัยคุกคาม

| ส่วนประกอบ | ความหมาย |
|------------|---------|
| Overview | ภาพรวม KPI, Heatmap เวลาการโจมตี, แหล่งข่าวกรอง |
| IOC Summary | สรุป IOC แยกตามประเภท, ความรุนแรง, แหล่งข้อมูล |
| Drill-down Reports | รายงานเจาะลึกตามประเภทภัย, แหล่งข้อมูล, กลุ่มเป้าหมาย |

### Operations Dashboard — สำหรับเจ้าหน้าที่ SOC

- Attack Time Heatmap — ช่วงเวลาที่ถูกโจมตีมากที่สุด (วัน × ชั่วโมง)
- Top Threat Types — ประเภทภัยคุกคามที่พบมากที่สุด
- Intelligence Sources — แหล่งข้อมูลที่ส่งข้อมูลมากที่สุด
- Severity Distribution — สัดส่วนระดับความรุนแรง

### IOC Data Lake — ค้นหาและกรอง IOC

ค้นหา IOC ทุกรายการ กรองตาม: ประเภท (IP/Domain/URL/Hash), ความรุนแรง, แหล่งข้อมูล, ช่วงเวลา พร้อมดูรายละเอียดเชิงลึกของแต่ละ IOC รวมถึงกราฟความสัมพันธ์

### Action Center — จัดการตอบสนอง

ระบบสร้าง Action สำหรับ IOC ที่มีความเสี่ยงสูงโดยอัตโนมัติ เจ้าหน้าที่สามารถ: มอบหมายงาน, เพิ่ม Analyst Note, บล็อก IP, ทำเครื่องหมาย False Positive

### CVE Intelligence — ติดตามช่องโหว่

ติดตามช่องโหว่ CVE ที่เกี่ยวข้อง เชื่อมโยงกับ IOC ที่พบในระบบ

## External Sharing API — แบ่งปันข้อมูลกับหน่วยงานพันธมิตร

ระบบเปิด API ให้หน่วยงานภายนอกเข้าถึงข้อมูลข่าวกรองได้ ผ่านการยืนยันตัวตนด้วย API Key:

| ความสามารถ | รายละเอียด |
|------------|-----------|
| ดูข้อมูล IOC | ค้นหา, กรอง, ดูรายละเอียด IOC |
| ดูความสัมพันธ์ | กราฟความสัมพันธ์ระหว่าง IOC |
| ติดตามการเปลี่ยนแปลง | Change Feed สำหรับ sync ข้อมูลแบบ incremental |
| ส่งข้อมูลเข้า | ส่ง IOC หรือ Event เข้าระบบ (ทีละรายการหรือ bulk) |
| Export ข้อมูล | ส่งออกข้อมูลในรูปแบบต่างๆ |

พันธมิตรแต่ละรายมี permission แยกกัน — สามารถกำหนดได้ว่าเห็นข้อมูลอะไรบ้าง

## ความปลอดภัย

| มาตรการ | รายละเอียด |
|---------|-----------|
| HTTPS | เข้าถึง Dashboard ผ่าน HTTPS เท่านั้น (Nginx reverse proxy) |
| Authentication | เข้าสู่ระบบด้วย username/password หรือ SSO |
| API Key | External API ต้องใช้ API Key ทุกครั้ง |
| Network Isolation | AI Service และ Database ผูกกับ localhost ไม่เปิดตรงจากภายนอก |
| Docker Container | แต่ละ service แยก container มี resource limit (CPU 4 core, RAM 6GB) |
| Permission | พันธมิตรแต่ละรายมี permission แยก ควบคุมการเข้าถึงข้อมูล |
